# Version Control & Git Usage

How developers use version control systems — especially Git — including tools, workflows, and attitudes.

## About this survey

This survey examines developers' use of version control, starting with a screener on how often and how long they have used it and which system and hosting platform they rely on. It documents workflow behaviors including how respondents interact with Git, commit and push frequency, branching strategy, commit-message habits, merge versus rebase, conflict resolution, pull requests, and code review, along with the specific Git and platform features they use. Attitudes are gathered through an agreement Likert battery about confidence and practices and a per-item importance matrix over hosting-platform criteria, plus a top-three ranking of reasons for using version control. Further sections cover self-rated skill, learning path, repository security, hosting model, satisfaction, blocking problems, overall satisfaction, recommendation likelihood, and switching intent. Question types include single-choice, multi-select, Likert scales, an importance matrix, a ranking item, and numeric free-text. The survey ends with a demographic block capturing role, professional experience, employment, company size, industry, age, gender, race and ethnicity, education, U.S. region, and urbanicity.
