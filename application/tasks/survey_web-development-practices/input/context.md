# Web Development Practices

How people build for the web: the frameworks, languages, tooling, and engineering practices they use and their satisfaction with them.

## About this survey

This survey profiles the respondent's web-development role, experience, and stack focus, then maps the front-end frameworks (React, Vue, Angular, Svelte, Next.js, Astro, HTMX, and more), back-end languages, styling approaches, hosting platforms, CMS/site builders, version control, deployment cadence, and engineering practices they use. It measures testing approach and tools, priorities on accessibility/performance/SEO/security/responsive design, AI-tool usage and estimated AI-assisted code share, browser testing, and how they keep skills current, via single-choice, multi-select, Likert agreement and priority batteries, and a satisfaction matrix split per item. It closes with a top-3 challenges ranking, a likelihood-to-recommend-the-career score, overall satisfaction, framework-adoption intent, and an open item. Question types include single-choice, multi-select, Likert scales, matrices split per item, a ranking, and numeric free-text. Demographics captured include age, gender, race/ethnicity, income, education, employment, years of professional experience, company size, work arrangement, U.S. region, and urban/suburban/rural.
